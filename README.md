## 初期设计简稿

**功能列表**

1. 语音助手：参考小布 AI，接入 ChatGPT，智能问答
2. 健康管理：提醒用药时间，病史和就诊记录（看病时给医生出示）
3. 老年圈子：微论坛，微社区，分享生活，评论互动
4. 娱乐资讯：短视频和新闻资讯（集成到一页）（服务端存储视频和文章，后台表单上传视频和录入文章）
5. 安全助手：紧急拨打电话求助，由家人录入电话数据
6. 社区活动：管理员发布活动，用户报名参加
7. 生活工具：天气预报、生活实用教程（出行、购物、娱乐）、智慧食谱（GPT 推荐）、宝宝取名、代沟分析、笑话大全、养生资讯

**TabBar**

1. 首页（紧急求助、健康管理、天气预报、工具Group）
2. 社区（社区活动（后台发布）、老年圈子）
3. 语音助手（图标，ChatGPT）
4. 资讯（实用教程、养生资讯、短剧（非三方接口，后台系统表单上传录入））
5. 我的（我的个人信息、我的收藏（Tab 文章、视频、短剧）、关于软件）

1. 研究背景与目标
伴随着老龄化社会的加剧，老年人对智能助手的需求日益增加。然而，目前市场上现有产品在功能和易用性上存在诸多不足。本研究的目标是设计一款集成多种实用功能的智能生活助手，充分利用鸿蒙操作系统的技术优势，提供便捷的智能服务。
2. 功能设计过程
在功能设计阶段，研究将通过需求调研和用户访谈，明确老年人在日常生活中最迫切的需求，并基于此制定功能列表。具体过程如下：
（1）语音助手
设计初期，将重点关注语音助手的功能开发。参考小布 AI，接入 ChatGPT 技术，确保其能够进行智能问答。通过自然语言处理，老年用户可以通过简单的语音指令获取信息，进行日常查询，如天气、新闻等。这一功能的设计考虑到老年人对文字输入的困难，力求提供流畅、友好的交互体验。
（2）健康管理
健康管理模块是智能助手的核心功能之一。通过调研，发现老年人对用药提醒和病史记录的需求较为迫切。因此，该模块将提供用药时间的提醒功能，并允许用户记录病史与就诊信息，以便在看病时便于向医生展示。这一功能的设计需要确保信息的安全性与隐私保护，采用数据加密技术保障用户数据安全。
（3）老年圈子
为了满足老年人社交需求，设计了“老年圈子”模块，包括微论坛和微社区功能。用户可以在此分享生活经验、发表评论，促进互动。该模块的设计目标是营造一个积极、温馨的社交氛围，帮助老年人缓解孤独感，提高社交参与感。
（4）娱乐资讯
该模块将整合短视频和新闻资讯。服务端存储视频和文章，后台支持管理员上传内容，确保信息的时效性与多样性。通过简化内容获取流程，用户可以轻松浏览感兴趣的娱乐内容，丰富日常生活。
（5）安全助手
安全性是老年人使用智能助手的重要考量。该模块将设定紧急拨打电话求助的功能，由家人预先录入紧急联系人电话。在紧急情况下，用户可以通过简单的语音指令快速联系到家人或医疗机构，增强安全保障。
（6）社区活动
为了促进老年人参与社区活动，设计了社区活动模块，管理员可以发布活动信息，用户可通过助手报名参加。这一功能旨在鼓励老年人走出家门，参与社区生活，增强归属感。
（7）生活工具
最后，生活工具模块将提供天气预报、实用教程、智慧食谱等功能。根据老年人的生活习惯和需求，开发多项实用功能，如出行指导、购物建议、娱乐推荐等，确保一站式服务。
